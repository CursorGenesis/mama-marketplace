# 📊 Настройка таблицы учёта агентов и магазинов

**Цель:** автоматизированный учёт оборотов, комиссий агентам, премий за поставщиков. Без ручного пересчёта.

**Уровень автоматизации:** 2 (полуавтомат) — данные о заказах поступают автоматически через код платформы, ты заполняешь только справочники (агенты/магазины/поставщики) + отмечаешь выплаты.

---

## 🔧 Что уже работает

В `src/lib/telegram.js:82` функция `sendToGoogleSheet` автоматически отправляет в твою Google Sheet каждый заказ со всеми данными:
- Номер заказа
- Поставщик (ID и название)
- Покупатель (ФИО, телефон, email, адрес)
- Магазин (если указан)
- Товары (JSON)
- Сумма заказа
- Комиссия платформы 5%
- Реф-код агента
- Комиссия агента 1%
- Начисленные монетки

**Ссылка на таблицу:** см. `NEXT_PUBLIC_GOOGLE_SHEET_URL` в `.env.local`.

Эти данные попадают в **Лист 1 «Заказы»** (или как он у тебя назван).

## 📋 Что будем добавлять — 4 новых листа

### Лист 2: «Агенты» — справочник агентов

Колонки:

| Колонка | Значение | Тип |
|---|---|---|
| A | ID агента (например AGT-001) | Текст |
| B | ФИО | Текст |
| C | Телефон | Текст |
| D | Основная компания (Шоро, Coca-Cola…) | Текст |
| E | Район работы | Текст |
| F | Оформление (Карта / Патент / ИП) | Текст |
| G | Дата старта | Дата |
| H | Статус (🟢 Активен / 🔴 Уволен) | Текст |

Ты заполняешь вручную при найме агента. 1 минута на одного.

---

### Лист 3: «Магазины» — кого привёл какой агент

Колонки:

| Колонка | Значение | Тип |
|---|---|---|
| A | ID магазина (SHP-001) | Текст |
| B | Название | Текст |
| C | Владелец | Текст |
| D | Телефон | Текст |
| E | Email (для связи с заказами) | Текст |
| F | **ID куратора** (AGT-001 или SHP-XXX) | Текст |
| G | Тип куратора (Агент / Реферал) | Текст |
| H | Дата подключения | Дата |

Заполняется при подключении магазина.

**Важно:** колонка E (Email) — ключ связи с Листом «Заказы», куда платформа пишет `buyerEmail`.

---

### Лист 4: «Поставщики» — премии агентам

Колонки:

| Колонка | Значение | Тип / формула |
|---|---|---|
| A | ID поставщика | Текст |
| B | Название | Текст |
| C | Категория | Текст |
| D | **Уровень (A/B/C/D)** | Текст |
| E | Дата подключения | Дата |
| F | **Кто привёл** (ID агента) | Текст |
| G | **Премия агенту** | Формула (см. ниже) |
| H | Статус премии (1 мес / 2 мес / выплачено) | Текст |
| I | Дата выплаты премии | `=E2+60` |

**Формула для колонки G (премия агенту по уровню):**
```
=IFS(D2="A";10000;D2="B";7000;D2="C";5000;D2="D";3000;TRUE;0)
```

Пишешь в ячейку G2, протягиваешь вниз. Премия автоматически подставится по уровню.

---

### Лист 5: «Выплаты агентам» (автомат за месяц)

Это главный лист — сюда попадают итоги за каждый месяц автоматически.

Колонки:

| Колонка | Значение | Формула |
|---|---|---|
| A | Месяц (06.2026) | Текст |
| B | ID агента | Текст |
| C | **Оборот магазинов агента** | Формула |
| D | **Комиссия агента 1%** | `=C2*0.01` |
| E | **Премии за поставщиков (закрытые в этом месяце)** | Формула |
| F | **Итого к выплате** | `=D2+E2` |
| G | Дата выплаты | Дата |
| H | Статус (Оплачено / Ждёт) | Текст |

### 📐 Формула для C (оборот магазинов агента за месяц)

```
=SUMIFS(
  Заказы!L:L,                          ← колонка "total" из листа заказов
  Заказы!I:I, VLOOKUP(B2; Магазины!E:F; 2; FALSE),   ← email магазина → куратор
  Заказы!A:A, ">=" & DATE(2026; 6; 1),  ← начало месяца
  Заказы!A:A, "<=" & DATE(2026; 6; 30)   ← конец месяца
)
```

*Точные номера колонок зависят от того как Google Apps Script пишет данные. Настраивается один раз.*

### 📐 Формула для E (премии за поставщиков в этом месяце)

```
=SUMIFS(
  Поставщики!G:G,            ← сумма премий
  Поставщики!F:F, B2,         ← агент = текущий
  Поставщики!H:H, "выплачено", ← статус
  Поставщики!I:I, ">=" & DATE(2026; 6; 1),  ← месяц выплаты
  Поставщики!I:I, "<=" & DATE(2026; 6; 30)
)
```

## ⏱ Пошаговая инструкция — 30 минут один раз

### Шаг 1 (5 мин). Открыть существующий Google Sheet

1. Зайди в Google Drive.
2. Найди таблицу куда приходят заказы (по ссылке из `NEXT_PUBLIC_GOOGLE_SHEET_URL`).
3. Открой её. Посмотри **точные названия колонок** в листе «Заказы» (они зависят от Apps Script который ты настраивала).
4. Запиши буквы колонок: где `buyerEmail`, где `total`, где дата.

### Шаг 2 (10 мин). Создать новые листы

1. Внизу таблицы плюсик «+» для добавления листа.
2. Создай 4 листа с именами: **«Агенты»**, **«Магазины»**, **«Поставщики»**, **«Выплаты»**.
3. В каждом заполни **шапку** (первую строку) согласно таблицам выше.

### Шаг 3 (10 мин). Вставить формулы

1. В листе **«Поставщики»**, ячейка G2 → вставь формулу для премии (см. выше).
2. В листе **«Выплаты»** — формулы для колонок C, D, E, F.
3. **Замени** в формулах буквы колонок на реальные из твоей таблицы заказов (которые записала в шаге 1).

### Шаг 4 (5 мин). Протестируй

1. Добавь в «Агенты» тестовую запись (AGT-TEST, любое ФИО).
2. Добавь в «Магазины» тестовый магазин с ID куратора = AGT-TEST и любым email.
3. Проверь есть ли заказ с таким email в «Заказы».
4. В «Выплаты» добавь строку с AGT-TEST, проверь что оборот подтянулся.

Если подтянулось — готово.

---

## 📊 Что ты получишь

### Раз в месяц (5-10 минут):
1. Открываешь Лист «Выплаты».
2. Видишь сколько должна агентам.
3. Отмечаешь оплату.
4. Лист «Поставщики» — отмечаешь статус премий (2 мес прошли — «выплачено»).

### Постоянно (автоматически):
- Каждый новый заказ пополняет лист «Заказы» → автоматически пересчитывает обороты всех агентов.
- Каждая подключённая магазин (после внесения в Лист «Магазины» с указанием куратора) — включается в статистику своего агента.

## ⚠️ Ограничения уровня 2

1. **Не работает для реферальных магазинов.** У нас реферальной программы между магазинами нет — значит ок.
2. **Не сохраняет историю статусов заказа** — для этого надо отдельный лист. Пока не критично.
3. **При росте > 1000 магазинов** Google Sheets начнёт тормозить — в этот момент переходим на уровень 3 (все данные только в платформе, панель в `/admin`).
